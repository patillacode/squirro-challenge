Hello, looking for the API?

  • /user/<username> to get the user with that <username>
  • /user to POST a user to enter into the DB (username, key)
  • /doc/doc_id/username/key to GET a document from the DB (username and key are credentials)
  • /doc to POST a doc to enter it into the DB (doc_id, doc_text, username, key)
  • POST should always be with json format:
  • doc POST example: '{"doc_id": "test_doc_2", "doc_text": "Continental does not have an office in Zurich.", "username" : "test", "key":"47CB9661-520D-4674-B79B-B0A1FD3805F1"}'
  • user POST example: '{"username" : "test2", "key":"47CB9661-520D-4674-B79B-B0A1FD3805F1"}'
